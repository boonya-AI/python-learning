很多人一提量化交易就犯怵，觉得全是复杂公式和代码，离自己特别远。但其实量化的本质特别简单：把 “怎么买卖” 的想法，变成电脑能执行的规则。今天就用最通俗的话，顺着 “想法→策略→程序→工具” 的路子，把量化交易的基本框架拆透，最后再结合真实代码举例，保证新手也能听明白。

## 第一步：先有简单想法
量化交易的起点，从来不是高大上的买卖逻辑，而是一个 “能一句话讲明白” 的买卖想法。就像咱们平时炒股，可能会说 “股价跌穿 5 日均线就卖，站上 20 日均线就买”—— 这就是最朴素的量化想法，也是很多经典策略的雏形。

我见过不少新手一开始就想搞多因子模型深度学习，结果逻辑绕了八圈，最后连个简单的买卖信号都写不出来。其实不如先从 “看得见、摸得着” 的想法入手，比如：

均线交叉：快线（短期均线）上穿慢线（长期均线）买，下穿卖；

价格突破：股价超过近 20 天最高点买，跌破近 20 天最低点卖；

成交量配合：股价涨 + 成交量放大买，股价跌 + 成交量放大卖。

大家最常见的“双均线策略”，核心想法就是最基础的 “均线交叉”—— 找两条不同周期的均线，金叉买、死叉卖。你别觉得这想法太简单，量化的关键不是 “想法多高深”，而是想法能不能落地。

## 第二步：把想法变成策略
有了想法，下一步就得把它细化成规矩—— 不然你说 “金叉买入”，电脑哪知道用哪两条均线，买多少？没现金了怎么办？这一步就像把模糊的灵感变成清晰的操作手册，咱们以上面提到的双均线策略为例，拆成 3 个关键问题：

1、定判断标准：用哪两条均线？怎么算？
想法里的 “快线、慢线” 太模糊，得给个具体数字。比如咱们选 “17 天快线” 和 “27 天慢线”—— 为啥是这两个数？不是瞎蒙的，可能是回测时发现，这两个周期在目标股票上的胜率、收益更均衡（你也可以改成 5 和 20、10 和 30，关键是 “定死一个标准，别来回变”）。

而且均线怎么算也得说清：比如用 “收盘价的平均值”，不是开盘价也不是最高价，这样电脑才知道该拿哪些数据计算。

2、定买卖细节：买多少？卖多少？
光判断该买还不够，得说清 “买多少”。比如设定 “每次买 10000 块钱的量”，电脑会根据当前股价算具体股数（比如股价 20 块，10000 块就能买 500 股），而且得符合股票 “100 股起买、整数倍” 的规则，不能买 150 股这种无效数量。

卖的时候也一样：要么 有多少卖多少，要么 卖一半留一半，咱们先选简单的 清仓卖—— 手里有多少股，死叉时就全卖掉，不用纠结 卖多少。

3、完善细节：补充买卖规则
做交易最怕乱下单，比如钱不够还买、前一笔单没成交又下单。所以得加些禁止规则，比如：

没现金别买：可用资金不够 10000 块，就算出金叉也不下单；

有未成交单别买：之前下的买入单还没成交，先别发新单，避免 重复下单导致买多了；

非交易时间别动：早上 9 点前、下午 3 点后，就算出信号也不操作，避免废单。

到这一步，一个 “想法” 就变成了 “可执行的策略”：

用 17/27 日均线算金叉死叉，每次买 10000 块的量、清仓卖，没现金 / 有未成交单 / 非交易时间不下单 —— 规则越清晰，后面写代码越简单。

## 第三步：找量化实盘软件
代码写好了，总得有个地方跑吧？不能只在自己电脑上算 “模拟盘”，还得能连真实行情、连股票账号，这时候就需要专业的量化软件，QMT 就是最常用的工具之一。

为啥选 QMT？3 个关键原因，咱们结合之前的策略说：

1、不用自己搭系统，直接用现成功能
比如咱们需要拿行情数据算均线，发下单指令—— 这些功能不是随便写个 Python 脚本就能实现的，QMT 已经帮你对接好了券商的行情接口和交易接口。

你不用自己琢磨怎么获取获取的实时收盘价，怎么查我的股票账号有多少钱，直接调用 QMT 的现成函数就行：比如用 “C.get_history_data” 拿历史收盘价，用 “get_trade_detail_data” 查账号余额，省了大量 “底层工作”，专注写策略逻辑就行。

2、支持实盘级风控，不怕乱下单
QMT 本身就带了很多风控功能，能查未成交委托：代码里可以直接调用函数，查之前的单有没有成交，没成交就暂停； 能区分账号类型：比如普通股票账号和两融账号的 “买卖代码” 不一样（股票买是 23，两融买是 33），QMT 能自动识别账号类型，填对代码，不用你手动改；

3、支持双语言，可回测
QMT 支持 Python 和 VBA 两种语言，咱们用 Python 写的双均线代码，直接复制到 QMT 的编辑器里就能跑，不用改太多。而且 QMT 有 “策略回测” 功能 —— 比如用过去 5 年的行情，先测测 “17/27 均线” 的策略到底赚不赚钱、最大亏多少，再决定要不要实盘，不用上来就用真金白银试错。

4、极速交易，延时低
QMT支持高度灵活的策略编写与回测，全内存交易，单笔延时小于1毫秒。这种极致的快，正好满足做量化、高频交易的人对速度的高要求

 #跳过历史k线
 if not C.is_last_bar():
  return
 now = datetime.datetime.now()
 now_time = now.strftime('%H%M%S')

  return
 account = account[0]
 available_cash = int(account.m_dAvailable)
 #如果有未查到委托 查询委托
 if A.waiting_list:
  found_list = []
  orders = get_trade_detail_data(A.acct, A.acct_type, 'order')
  for order in orders:
   if order.m_strRemark in A.waiting_list:
    found_list.append(order.m_strRemark)

 pre_line2 = np.mean(close_list[-A.line2-1: -1])
